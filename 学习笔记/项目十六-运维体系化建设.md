# 项目十六: 运维体系化建设

## 项目概述
学习如何建立标准化的运维体系，通过识别运维对象、属性、关系和场景，构建系统化的运维管理模型。

## 核心知识点

### 1. 标准化四步法
标准化 = 识别对象 + 识别属性 + 识别关系 + 识别场景

### 2. 基础设施层面标准化
- **识别对象**：服务器、网络设备、IDC 机房、机柜、存储、配件
- **识别属性**：
  - 服务器：型号（CPU/内存/硬盘）、地址（内网/MAC/带外/外网）、维保（厂商/日期/SN）
- **识别关系**：服务器 → 机柜 → 机房（网络拓扑图）
- **识别场景**：采购、入库、安装、配置、上下线、维修

### 3. 应用层面标准化
- **识别对象**：微服务应用（由业务架构师确定）
- **识别属性**：
  - 元数据：应用名、Owner、所属业务、是否核心链路
  - 代码属性：编程语言、Git 地址、分支结构
  - 部署模式：软件包名、容器（Tomcat/JBoss）
  - 目录信息：脚本目录、日志目录、应用包目录
  - 运行脚本：启停脚本、监控检测脚本
  - 运行参数：端口、JVM 参数
- **识别关系**：
  - 应用与基础设施：资源、VIP、DNS
  - 应用与应用：全链路拓扑
  - 应用与基础组件：缓存、消息队列、数据库
- **识别场景**：CI/CD、扩容缩容、容量评估、压测、限流降级

### 4. 运维体系化建设要点
- 所有运维操作都应针对明确的对象
- 脱离对象的运维操作没有意义
- 标准化是自动化的前提
- CMDB（配置管理数据库）是标准化的载体

## 面试高频题

### Q1: 如何理解运维标准化的意义？
**答：** 标准化是运维体系的基础。通过识别对象（服务器、应用）、属性（配置信息）、关系（依赖拓扑）、场景（操作流程），建立统一的运维模型。标准化是自动化的前提，没有标准化就无法实现自动化运维。

### Q2: CMDB 在运维体系中的作用？
**答：** CMDB（配置管理数据库）是标准化的载体，存储所有运维对象的属性和关系。作用：① 资产管理；② 变更管理；③ 故障定位（通过拓扑关系快速定位影响范围）；④ 自动化运维的基础数据源。

### Q3: 应用标准化需要包含哪些信息？
**答：** 元数据（名称、Owner、业务线）、代码信息（Git 地址、语言）、部署信息（包名、容器类型）、目录结构（日志、脚本、数据）、运行参数（端口、JVM）、启停脚本、监控脚本、依赖关系（数据库、缓存、消息队列）。

## 学习心得
- 运维标准化是从小作坊式运维走向体系化运维的第一步
- "识别对象 → 识别属性 → 识别关系 → 识别场景"四步法适用于任何运维领域
- CMDB 是运维体系的核心基础设施，维护成本高但价值大
- 应用标准化需要开发和运维共同参与，单方面推动很难成功
