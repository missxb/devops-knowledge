## 一、Docker 基础概念

### 1.1 什么是 Docker？
Docker 是一个开源的容器化平台，让开发者可以把应用及其依赖打包成轻量级的容器。

### 1.2 核心概念
| 概念 | 说明 |
|------|------|
| **镜像 (Image)** | 模板，类似面向对象的类 |
| **容器 (Container)** | 镜像的运行实例，类似对象 |
| **仓库 (Repository)** | 存放镜像的地方 |

### 1.3 Docker vs 虚拟机
| 特性 | Docker | 虚拟机 |
|------|--------|--------|
| 启动速度 | 秒级 | 分钟级 |
| 资源占用 | 极小 | 较大 |
| 隔离性 | 进程级 | 完整隔离 |
| 操作系统 | 共享内核 | 独立内核 |

## 二、Docker 核心命令

### 2.1 镜像操作
```bash
# 拉取镜像
docker pull nginx:latest

# 查看镜像
docker images

# 构建镜像
docker build -t myapp:v1 .

# 删除镜像
docker rmi nginx:latest

# 推送镜像
docker push myapp:v1
```

### 2.2 容器操作
```bash
# 运行容器
docker run -d -p 8080:80 --name mynginx nginx

# 查看运行中的容器
docker ps

# 查看所有容器
docker ps -a

# 停止/启动容器
docker stop mynginx
docker start mynginx

# 删除容器
docker rm mynginx

# 进入容器
docker exec -it mynginx /bin/bash

# 查看容器日志
docker logs -f mynginx
```

### 2.3 常用参数
- `-d`: 后台运行
- `-p`: 端口映射（主机端口:容器端口）
- `-v`: 挂载卷（主机路径:容器路径）
- `--name`: 指定容器名称
- `-e`: 环境变量
- `--restart`: 自动重启策略

## 三、Dockerfile 最佳实践

### 3.1 基本结构
```dockerfile
# 基础镜像
FROM node:18-alpine

# 维护者信息
LABEL maintainer="xiaobai@devops.com"

# 设置工作目录
WORKDIR /app

# 复制文件
COPY package*.json ./

# 安装依赖
RUN npm install

# 复制源码
COPY . .

# 暴露端口
EXPOSE 3000

# 启动命令
CMD ["node", "index.js"]
```

### 3.2 最佳实践
1. **减少镜像层数**：合并 RUN 指令
2. **使用 .dockerignore**：排除不需要的文件
3. **多阶段构建**：减小镜像体积
4. **使用特定版本标签**：避免使用 latest
5. **清理缓存和临时文件**

### 3.3 多阶段构建示例
```dockerfile
# 构建阶段
FROM golang:1.20 AS builder
WORKDIR /app
COPY . .
RUN go build -o myapp

# 运行阶段
FROM alpine:latest
WORKDIR /app
COPY --from=builder /app/myapp .
CMD ["./myapp"]
```

## 四、Docker 网络

### 4.1 网络模式
| 模式 | 说明 |
|------|------|
| **bridge** | 默认模式，容器间可通信 |
| **host** | 与主机共享网络 |
| **none** | 无网络 |
| **container** | 共享另一个容器的网络 |

### 4.2 自定义网络
```bash
# 创建网络
docker network create mynetwork

# 运行容器到指定网络
docker run -d --network mynetwork --name web nginx

# 容器间通信（使用容器名）
docker exec -it web ping db
```

## 五、Docker 存储

### 5.1 数据卷 (Volume)
```bash
# 创建数据卷
docker volume create mydata

# 挂载数据卷
docker run -v mydata:/app/data nginx

# 查看数据卷
docker volume ls
docker volume inspect mydata
```

### 5.2 绑定挂载
```bash
# 挂载主机目录
docker run -v /host/path:/container/path nginx
```

## 六、Docker Compose

### 6.1 docker-compose.yml 示例
```yaml
version: '3.8'

services:
  web:
    image: nginx
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
    depends_on:
      - app
  
  app:
    image: node:18
    working_dir: /app
    volumes:
      - .:/app
    command: npm start
    depends_on:
      - db

  db:
    image: mysql:8
    environment:
      MYSQL_ROOT_PASSWORD: secret
    volumes:
      - dbdata:/var/lib/mysql

volumes:
  dbdata:
```

### 6.2 常用命令
```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down

# 重新构建
docker-compose up -d --build
```

## 七、Harbor 私有镜像仓库

> 参考项目一：Harbor 多实例高可用共享存储

### 7.1 Harbor 核心功能
- 镜像存储与管理
- 镜像签名与验证
- 访问控制
- 漏洞扫描
- 镜像复制（高可用）

### 7.2 基本使用
```bash
# 登录 Harbor
docker login harbor.example.com -u admin

# 标记镜像
docker tag myapp:latest harbor.example.com/library/myapp:latest

# 推送镜像
docker push harbor.example.com/library/myapp:latest

# 拉取镜像
docker pull harbor.example.com/library/myapp:latest
```
