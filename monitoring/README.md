# 监控与告警

## 监控体系

### 四大黄金信号
1. **延迟** - 请求处理时间
2. **流量** - 系统负载
3. **错误** - 错误率
4. **饱和度** - 资源使用率

## 工具栈

### Prometheus
- 时序数据库
- 服务发现
- 告警规则
- 多维数据模型

### Grafana
- 可视化面板
- 灵活的查询语言
- 丰富的插件生态

### 日志管理
- **ELK Stack** (Elasticsearch, Logstash, Kibana)
- **Loki** - 轻量级日志聚合
- **Fluentd / Fluent Bit** - 日志收集

### APM (应用性能监控)
- **Jaeger / Zipkin** - 分布式追踪
- **SkyWalking** - APM 和追踪

## 告警策略

### 告警级别
- **P0** - 严重，立即处理
- **P1** - 高优先级，1小时内
- **P2** - 中等，4小时内
- **P3** - 低优先级，24小时内

### 告警渠道
- 邮件
- 短信
- 即时通讯 (钉钉、企业微信、Slack)
- 电话
