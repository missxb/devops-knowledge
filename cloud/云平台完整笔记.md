## 三、VPC 网络设计

### 3.1 企业级 VPC 架构
```
                    ┌─────────────────────────────────┐
                    │         Internet                 │
                    └──────────┬──────────────────────┘
                               │
                    ┌──────────┴──────────────┐
                    │    NAT Gateway / SLB     │
                    └──────────┬──────────────┘
                               │
              ┌────────────────┼────────────────┐
              │                │                │
    ┌─────────┴─────────┐    │    ┌───────────┴──────────┐
    │  Public Subnet     │    │    │  Private Subnet      │
    │  (Web层)           │    │    │  (App层/DB层)         │
    │  ┌───┐ ┌───┐      │    │    │  ┌───┐ ┌───┐         │
    │  │Nginx│ │Nginx│   │    │    │  │App│ │DB │         │
    │  └───┘ └───┘      │    │    │  └───┘ └───┘         │
    └────────────────────┘    │    └──────────────────────┘
                              │
                    ┌─────────┴─────────┐
                    │  Management Subnet │
                    │  (跳板机/监控)      │
                    └───────────────────┘
```

### 3.2 子网规划
| 子网 | CIDR | 用途 | 路由 |
|------|------|------|------|
| Public | 10.0.1.0/24 | SLB、Nginx | Internet Gateway |
| App | 10.0.2.0/24 | 应用服务器 | NAT Gateway |
| DB | 10.0.3.0/24 | 数据库 | 无出公网 |
| Mgmt | 10.0.4.0/24 | 跳板机、监控 | NAT Gateway |

## 四、混合云架构

### 4.1 连接方式
| 方式 | 带宽 | 延迟 | 成本 | 适用场景 |
|------|------|------|------|----------|
| VPN | 低 | 高 | 低 | 临时/测试 |

```
│                    │     │  AI/ML 训练              │
└────────────────────┘     └─────────────────────────┘
```

## 五、云上成本优化

### 5.1 计费模式
| 模式 | 折扣 | 适用场景 |
|------|------|----------|
| 按量付费 | 100% | 测试/突发 |
| 包年包月 | 30-60% | 稳定业务 |
| 预留实例 | 40-70% | 长期使用 |
| 竞价实例 | 50-90% | 容错任务（批处理/CI） |

### 5.2 成本优化清单
- [ ] 清理僵尸资源（未挂载的云盘、闲置EIP）
- [ ] 合理选择实例规格（不要过度配置）
- [ ] 使用自动伸缩（ASG/HPA）
- [ ] 数据分层存储（热/温/冷/归档）
- [ ] 预留实例覆盖稳定负载
- [ ] 竞价实例用于 CI/CD 和批处理

## 六、面试高频题

### Q1: 公有云、私有云、混合云的区别？
**答：** 公有云是多租户共享资源，按需付费（如AWS/阿里云）；私有云是企业自建独享资源，安全性高但成本大；混合云结合两者，核心数据放私有云，弹性负载放公有云。企业通常选择混合云，兼顾安全和弹性。

### Q2: 如何选择云厂商？
**答：** 考虑五个维度：① 全球覆盖（业务区域）；② 服务完整性（需要的服务是否都有）；③ 生态和社区（文档、SDK、社区活跃度）；④ 成本（对比同规格价格）；⑤ 合规（数据主权、等保要求）。国内首选阿里云，出海首选AWS。

### Q3: 云上安全和传统安全有什么不同？
**答：** 云上安全采用"责任共担模型"：云厂商负责底层基础设施安全（物理、网络、虚拟化），用户负责上层安全（操作系统、应用、数据、访问控制）。用户需要关注：IAM 权限管理、网络安全组、数据加密、日志审计。

## 七、学习心得
- 先理解核心服务（计算/存储/网络/数据库），再学高级服务
- VPC 网络设计是基础，必须掌握子网、路由、安全组
- 混合云是企业常态，专线和 VPN 是必备技能
- 成本优化是持续工作，不是一次性的
- 多云策略避免厂商锁定，但增加复杂度
