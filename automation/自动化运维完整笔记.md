# 自动化运维完整学习笔记

## 一、Ansible - 无代理配置管理

### 1.1 核心特点
- **无代理 (Agentless)**：通过 SSH 连接，不需要在目标机器装软件
- **声明式**：描述期望状态，Ansible 确保达到
- **幂等性**：多次执行结果一致
- **YAML 语法**：Playbook 用 YAML 编写，易读易写

### 1.2 架构
```
Ansible 控制节点
       ↓ (SSH)
  ┌────┴────┬────────┬────────┐
  Node1    Node2   Node3   Node4
  (无需安装Agent)
```

### 1.3 核心概念
| 概念 | 说明 |
|------|------|
| Inventory | 主机清单，定义管理的主机 |
| Module | 功能模块（yum/copy/service/template等） |
| Playbook | 任务编排，YAML格式 |
| Role | 可复用的任务集合 |
| Handler | 由 notify 触发的条件任务 |
| Variable | 变量，支持多级优先级 |

### 1.4 Inventory 文件
```ini
# /etc/ansible/hosts
[webservers]
web01 ansible_host=192.168.1.10
web02 ansible_host=192.168.1.11

[dbservers]
db01 ansible_host=192.168.1.20

[all:vars]
ansible_user=deploy
ansible_ssh_private_key_file=~/.ssh/id_rsa
```

### 1.5 Playbook 示例
```yaml
# deploy-web.yml
---
- hosts: webservers
  become: yes
  vars:
    app_version: "1.2.3"
    app_port: 8080
  
  tasks:
    - name: 安装依赖
      yum:
        name: [java-11-openjdk, nginx]
        state: present

    - name: 部署应用包
      copy:
        src: "files/app-{{ app_version }}.jar"
        dest: /opt/app/app.jar
        mode: '0755'
      notify: restart app

    - name: 部署配置文件
      template:
        src: templates/app.conf.j2
        dest: /etc/app/app.conf
      notify: restart app

    - name: 确保服务运行
      systemd:
        name: myapp
        state: started
        enabled: yes

  handlers:
    - name: restart app
      systemd:
        name: myapp
        state: restarted
```

### 1.6 Role 目录结构
```
roles/
└── nginx/
    ├── tasks/main.yml
    ├── handlers/main.yml
    ├── templates/nginx.conf.j2
    ├── files/
    ├── vars/main.yml
    └── defaults/main.yml
```

### 1.7 常用命令
```bash
# 执行 Ad-hoc 命令
ansible all -m ping
ansible webservers -m shell -a "uptime"
ansible webservers -m yum -a "name=nginx state=present" --become

# 执行 Playbook
ansible-playbook deploy-web.yml
ansible-playbook deploy-web.yml --check        # 模拟执行
ansible-playbook deploy-web.yml --limit web01  # 只在web01执行
ansible-playbook deploy-web.yml -e "app_version=1.2.4"  # 传变量
```

## 二、Terraform - 基础设施即代码

### 2.1 核心特点
- **声明式**：描述期望的基础设施状态
- **多云支持**：AWS/阿里云/Azure/GCP 等
- **状态管理**：维护基础设施当前状态
- **执行计划**：变更前预览（plan）
- **幂等性**：多次 apply 结果一致

### 2.2 工作流程
```
编写 .tf 文件 → terraform init → terraform plan → terraform apply
                                                     ↓
                                               状态文件 (state)
                                                     ↓
                                               云资源创建/修改/删除
```

### 2.3 HCL 语法示例
```hcl
# provider.tf
terraform {
  required_providers {
    alicloud = {
      source  = "aliyun/alicloud"
      version = "~> 1.200"
    }
  }
  backend "oss" {
    bucket = "terraform-state"
    prefix = "prod"
    region = "cn-hangzhou"
  }
}

provider "alicloud" {
  region = "cn-hangzhou"
}

# main.tf
resource "alicloud_vpc" "main" {
  vpc_name   = "prod-vpc"
  cidr_block = "10.0.0.0/16"
}

resource "alicloud_vswitch" "app" {
  vpc_id     = alicloud_vpc.main.id
  cidr_block = "10.0.1.0/24"
  zone_id    = "cn-hangzhou-h"
}

resource "alicloud_instance" "web" {
  count                = 3
  instance_name        = "web-${count.index + 1}"
  image_id             = "centos_7_9_x64_20G_alibase_20230816.vhd"
  instance_type        = "ecs.c6.large"
  security_groups      = [alicloud_security_group.default.id]
  vswitch_id           = alicloud_vswitch.app.id
  system_disk_category = "cloud_essd"
  system_disk_size     = 40
}

# outputs.tf
output "web_ips" {
  value = alicloud_instance.web[*].private_ip
}
```

### 2.4 常用命令
```bash
terraform init      # 初始化，下载 provider
terraform plan      # 预览变更
terraform apply     # 执行变更
terraform destroy   # 销毁资源
terraform state list  # 查看管理的资源
terraform import alicloud_instance.web i-xxx  # 导入已有资源
terraform output    # 查看输出
```

### 2.5 State 管理最佳实践
1. **远程存储**：使用 OSS/S3 存储 state，不要存本地
2. **状态锁**：启用 DynamoDB/OSS 表锁防止并发修改
3. **分环境隔离**：每个环境独立 state
4. **敏感数据**：state 文件包含敏感信息，需要加密存储

## 四、面试高频题

### Q1: Ansible 和 Terraform 的区别？
**答：** Ansible 是配置管理工具，用于在已有服务器上安装软件、部署应用（无代理，SSH连接）。Terraform 是基础设施编排工具，用于创建和管理云资源（VPC/ECS/RDS等）。简单说：Terraform 管"创建服务器"，Ansible 管"服务器里装什么"。两者可以配合使用。

### Q2: 什么是幂等性？为什么重要？
**答：** 幂等性指多次执行同一操作，结果一致。例如"确保 nginx 已安装"，不管执行几次，结果都是 nginx 已安装。这很重要因为：① 网络故障可以安全重试；② 可以定期执行保持一致性；③ 减少"配置漂移"。

### Q3: Terraform state 文件有什么作用？
**答：** State 文件记录了 Terraform 管理的资源的实际状态（ID、属性等）。作用：① 映射配置文件和真实资源；② 检测变更（对比配置和实际状态）；③ 提升性能（缓存资源属性）。必须远程存储、启用锁、定期备份。

## 五、学习心得
- Ansible 上手快，适合中小规模，Playbook 语法要熟练
- Terraform 是多云管理利器，HCL 语法需要练习
- IaC 的核心价值是版本化、可重复、可审计
- 先用 Terraform 创建资源，再用 Ansible 配置，是经典组合
- State 管理是 Terraform 的关键，出问题很头疼
